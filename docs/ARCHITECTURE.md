# Architecture — Waypoint Travel Guides

Developer/agent orientation. The working rules (ship loop, continuity sweep, accuracy
standards) live in the auto-loaded `CLAUDE.md` (project) stacked on `~/.claude/CLAUDE.md`
(accuracy); content-authoring standards live in the `waypoint-guide-author` skill. This
file is durable architecture only — for the exact current shape prefer `git log` and the
`src/` tree over prose.

**Repo:** github.com/Carlob2499/Trip-Guides · **Live:** carlob2499.github.io/Trip-Guides/

## What it is

A static **Astro** site of curated, fact-checked, personalized travel guides — never
autogenerated from training data (see `CLAUDE.md` for the 4-property bar). The owner is
not a programmer: edit JSON → push → GitHub Actions rebuilds and deploys to GitHub Pages.
Live guides: **Korea**, **Denmark**.

**Stack:** Astro (static) · JSON content in `src/content/guides/` (validated by a Zod
discriminated union in `src/content.config.ts`) · per-feature CSS in `src/styles/` +
client modules in `src/scripts/` · self-contained features in `src/features/<name>/`
(Firebase live-sync is the reference) · GitHub Pages (base `/Trip-Guides`) · PWA (offline
via `public/sw.js`) · CI in `.github/workflows/`.

**Maps:** OpenStreetMap iframe by default; the Google renderer (`gmaps-render.js`)
self-boots to upgrade it only when `PUBLIC_GMAPS_KEY` is set.

## File map

- `src/content/guides/*.json` — the guides (the only per-country files).
- `src/content.config.ts` — the section schema (discriminated union of block types).
- `src/layouts/GuideLayout.astro` — the guide shell; imports styles + client modules.
  `src/components/blocks/*.astro` — one renderer per section type, dispatched by `Block.astro`.
- `src/pages/index.astro` — the hub (verified guides only). `og/[slug].png.ts`,
  `guides/[slug].gpx.ts`, `.ics.ts` — build-time generated assets per guide.
- `src/data/countries.mjs` — single source for accent / currency / IANA tz / ISO / capital /
  emergency numbers (imported by both TS and Node scripts).
- `src/lib/` — pure build-time helpers (exports, holidays, map-pins, themes, settle) + tests.
- `src/features/<name>/` — self-contained features (sealed dir, single `index.ts` public API).
- `scripts/` — guide generation (`generate-guide.ts` + Groq pipeline), enrichers
  (`lookup-place`, `search-commons`, `fetch-wikivoyage`), the weekly content audit
  (`audit/*`), and the new-guide scaffold.
- `.claude/skills/waypoint-guide-author/` — the content-authoring skill (verification and
  writing standards live here, not in this doc).

## Architecture decisions (the durable ones)

- **JSON, not a CMS/MDX** — the owner edits data, not markup; the schema is the guardrail.
- **Shared components are global; only JSON is per-country** — a component change hits every
  guide at once (intended).
- **New feature = its own folder** — `src/features/<name>/` with a single `index.ts` API;
  third-party SDKs behind a config gate + lazy import (Firebase / Google Maps are the pattern).
- **Exports are build-time endpoints** (`getStaticPaths` + GET), mirroring the OG image — no
  separate pre-build step, no runtime.
- **Base-path hrefs are explicit** — every internal `/`-href uses `import.meta.env.BASE_URL`.
- **Live data degrades gracefully** — currency/weather/holidays fetch-once, cache, and hide or
  fall back on failure; never break the page.
- **JSON encoding** — guide files are UTF-8; keep native-script text and `≈`/`⚠` glyphs intact.
  On Windows, `PYTHONUTF8=1` for any Python touching them.

## How to work with it

- **Content change:** edit the guide JSON, then run the continuity sweep + ship loop from
  `CLAUDE.md`. Fact edits follow the `waypoint-guide-author` skill.
- **New country:** add a row to `src/data/countries.mjs`, then scaffold/author (intake:
  `docs/NEW_GUIDE_INTAKE.md`).
- **New section type:** add a Zod member to `src/content.config.ts`, a renderer in
  `src/components/blocks/`, and wire it in `Block.astro`.
