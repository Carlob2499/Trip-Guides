## What it is

A static **Astro** site of curated, fact-checked, personalized travel guides —
never autogenerated from training data (see `CLAUDE.md` for the 4-property bar).
The owner is not a programmer: the workflow is edit JSON → push → GitHub Actions
rebuilds and deploys to GitHub Pages. Two live guides today: **Korea**, **Denmark**.

**Stack:** Astro (static) · JSON content in `src/content/guides/` (one file per
destination, validated by a Zod discriminated union in `src/content.config.ts`) ·
per-feature CSS in `src/styles/` + per-feature client modules in `src/scripts/` ·
GitHub Pages (base path `/Trip-Guides`) · PWA (installable, offline via
`public/sw.js`) · CI in `.github/workflows/deploy.yml`.

**Maps:** a lightweight OpenStreetMap iframe embed by default; the Google renderer
(`gmaps-render.js`) self-boots to upgrade it only when a `PUBLIC_GMAPS_KEY` is set.

## File map

- `src/content/guides/*.json` — the guides (the only per-country files).
- `src/content.config.ts` — the section schema (discriminated union of block types).
- `src/layouts/GuideLayout.astro` — the guide page shell; imports every style +
  client module. `src/components/blocks/*.astro` — one renderer per section type.
- `src/pages/index.astro` — the hub (verified guides only). `og/[slug].png.ts`,
  `guides/[slug].gpx.ts`, `.ics.ts` — build-time generated assets per guide.
- `src/data/countries.mjs` — single source for accent / currency / IANA tz / ISO /
  capital / emergency numbers, imported by both TS and Node scripts.
- `src/lib/` — pure build-time helpers (exports, holidays, map-pins, themes,
  lead-split) with unit tests.
- `scripts/` — guide generation (`generate-guide.ts` + Groq pipeline), enrichers
  (`lookup-place`, `search-commons`, `fetch-wikivoyage`), the weekly content audit
  (`audit/*`), holiday fetch, and the new-guide scaffold.
- `.claude/skills/waypoint-guide-author/` — the content-authoring skill (all the
  verification/writing standards live here, not in this doc).

## How to work with it

- **Content change:** edit the guide JSON, then run the continuity sweep and ship
  loop from `CLAUDE.md`. Fact edits go under the `waypoint-guide-author` skill.
- **New country:** add a row to `src/data/countries.mjs`, then scaffold/author.
- **New section type:** add a Zod member to `src/content.config.ts`, a renderer in
  `src/components/blocks/`, and wire it in `Block.astro`.
- **JSON encoding (critical):** guide files are UTF-8; keep native-script text and
  `≈`/`⚠` glyphs intact. On Windows, `PYTHONUTF8=1` for any Python touching them.
- **Standards, ship loop, gotchas:** in `CLAUDE.md` (auto-loaded) and the skill —
  not repeated here.

## Architecture decisions (the durable ones)

- **JSON, not a CMS/MDX** — the owner edits data, not markup; the schema is the guardrail.
- **Shared components are global; only JSON is per-country** — a component change
  hits every guide at once (intended).
- **Exports are build-time endpoints** (`getStaticPaths` + GET) mirroring the OG
  image, so they need no separate pre-build step and no runtime.
- **Base-path hrefs are explicit** — every internal `/`-href uses `import.meta.env.BASE_URL`.
- **Live data degrades gracefully** — currency/weather/holidays fetch-once, cache,
  and hide or fall back on failure; never break the page.
